有关于redis做验证码缓存的记录

本文总共733个字 | 您是第位看到它们的小伙伴

为什么需要用redis做分布式缓存？

缓存分为本地缓存和分布式缓存。以java为例，使用自带的map或者guava实现的是本地缓存，最主要的特点是轻量以及快速，生命周期随着jvm的销毁而结束，并且在多实例的情况下，每个实例都需要各自保存一份缓存，缓存不具有一致性。

使用redis或memcached之类的称为分布式缓存，在多实例的情况下，各实例共用一份缓存数据，缓存具有一致性。缺点是需要保持redis或memcached服务的高可用，整个程序架构上较为复杂。

Json对象转换

将key value 存放进redis中时，将value由实例对象转换为String或者object时遇到的了有关于json对象转换之间的问题。

json <– string Json.tostring()

Object <–json

反序列化时可以指定转化的类 XXX.class 但是我们必须保证XXX中有默认的无参构造函数，如果类中已有有参的构造函数或者自定义构造函数，必须再声明一个无参数空的构造函数。 （原因没弄懂，先记下）

记录

一开始想在jcaptcha插件的基础上，以key value的形式存放在redis中。
执行的过程中发现Jcaptcha生成的验证码字符串无法直接或者间接获得，只能获得captchaandlocal 类。而该类没有默认构造函数。所以无法在插件的基础上做缓存。所以最后决定不使用插件直接手写一个验证码生成，然后以键值的形式存放在redis中。
